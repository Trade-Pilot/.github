# Trade Pilot - 개발 로드맵

## 개발 전략

Trade Pilot은 **데이터 수집 → 시뮬레이션 → 가상거래 → 실거래** 순서로 점진적으로 개발합니다.

```
Phase 1          Phase 2          Phase 3          Phase 4
데이터 기반      전략 개발        실시간 검증      실전 투입
구축             및 검증          환경             자동화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Market          Agent           VirtualTrade     Trade
Service         Service         Service          Service
(3개월)         (2개월)         (2개월)          (2개월)
```

### 개요
Phase 5는 3개의 독립적인 서브 Phase로 구성되며, **순차적으로 진행**합니다.

**이유**: AI 전략은 연구 성격이 강하므로 별도 장기 프로젝트로 분리 권장

- 이유: 불확실성이 높고, 장기간 연구 필요

- [ ] Train/Validation/Test 분할 (70/15/15)

#### 5-3.4 모델 학습 (8주)
- [ ] 학습 환경 구성 (GPU 서버)
- [ ] 하이퍼파라미터 튜닝
- [ ] 학습 실행 (최소 100만 에피소드)
- [ ] 과적합 방지 (Early Stopping, Dropout)
- [ ] 학습 곡선 모니터링

#### 5-3.5 백테스팅 및 검증 (4주)
- [ ] Out-Sample 데이터 백테스팅
- [ ] 수동 전략 대비 성과 비교
- [ ] 리스크 분석 (MDD, 샤프 비율)
- [ ] 전략 해석 (Explainability)

#### 5-3.6 모델 배포 및 운영 (4주)
- [ ] 모델 서빙 시스템 구축
- [ ] 가상거래 검증 (2개월)
- [ ] A/B 테스트 (AI vs 수동 전략)
- [ ] 성과 모니터링
- [ ] 모델 재학습 파이프라인

### Phase 5-3 완료 조건
- ✅ AI 전략 백테스팅 승률 > 65%
- ✅ 수동 전략 대비 수익률 +20% 이상
- ✅ 가상거래 2개월 안정 운영
- ✅ 샤프 비율 > 2.5

### Phase 5-3 리스크
- **높은 불확실성**: AI 전략 성공 보장 없음
- **장기 연구 필요**: 최소 6개월 이상
- **높은 컴퓨팅 비용**: GPU 서버 필요
- **과적합 위험**: 백테스팅 성공 ≠ 실거래 성공

**권장**: Phase 5-3는 별도 연구 프로젝트로 분리하고, Phase 1~4 안정 운영에 집중

## 리스크 관리

상세한 리스크 대응 방안은 각 **[기능별 문서](features/)**를 참조하세요.

### 리스크 매트릭스

| 리스크 | 발생 가능성 | 영향도 | 우선순위 | 대응 |
|--------|------------|--------|----------|------|
| 시스템 장애 | Medium | Critical | P0 | HA 구성, RTO < 5분 |
| 데이터 품질 | Medium | High | P1 | Flat Candle 보정, 검증 로직 |
| API 장애 | High | Critical | P0 | Circuit Breaker, 재시도 |
| 전략 실패 | High | Critical | P0 | Walk-Forward, 가상거래 1개월 |
| 시장 급변 | Low | Critical | P1 | 변동성 모니터링, 긴급 청산 |
| 자금 손실 | Medium | Critical | P0 | 손실 한도, 긴급 중단 |
| 규제 변경 | Low | High | P2 | 분기 법률 자문, 백업 플랜 |
| API 키 유출 | Low | Critical | P1 | AES-256, IP 화이트리스트 |
| DB 유출 | Low | Critical | P1 | TDE, 침투 테스트 |

### 기술적 리스크
- **데이터 누락**: Flat Candle 자동 보정, 수집 성공률 > 99.9%
- **시스템 장애**: Kubernetes HA 구성, 자동 재시작, RTO < 5분, RPO < 1분
- **API 제한**: Rate Limiting 준수 (초당 10개), Exponential Backoff, Circuit Breaker

### 비즈니스 리스크
- **전략 실패**: Walk-Forward Analysis, 가상거래 1개월 필수 검증, 점진적 자금 투입
- **자금 손실**: 일일 3%/주간 7%/월간 15% 손실 한도, 긴급 중단 시스템
- **시장 급변**: 변동성 > 50% 시 자동 중지, 손절가 자동 설정 (-5%)
- **규제 변경**: 분기 1회 법률 자문, 규제 모니터링, 백업 플랜 (주식 시장 전환)

### 보안 리스크
- **API 키 유출**: AES-256-GCM 암호화, 접근 로그 기록, IP 화이트리스트
- **데이터베이스 유출**: TDE 암호화, SQL Injection 방지, 침투 테스트 (분기 1회)
- **2FA**: 거래소 계정, 관리자 계정 2단계 인증 필수

### 재난 복구 훈련 (DR Drill)
**분기 1회 필수 실시**:
- Q1: 시스템 장애 대응 (K3s 노드 다운)
- Q2: API 장애 대응 (거래소 API 다운)
- Q3: 보안 침해 대응 (API 키 유출)
- Q4: 블랙스완 대응 (시장 급락 -30%)

## 측정 지표 (KPI)

상세한 KPI 지표는 **[기능별 문서](features/)**를 참조하세요.

### Phase 1 (데이터 수집)
- 데이터 수집 성공률 > 99.9%
- 시스템 가동률 > 99.5%
- API 응답 시간 < 100ms (p95)
- Flat Candle 비율 < 5%

### Phase 2 (백테스팅)
- 전략 승률 > 60%
- 샤프 비율 > 1.5
- MDD < 20%
- 검증된 전략 수 > 3개

### Phase 3 (가상거래)
- 월평균 수익률 > 3%
- BTC 대비 초과 수익 > +3%
- MDD < 15%
- 일일 거래 실행 성공률 > 99%
- 리스크 한도 준수율 100%

### Phase 4 (실거래)
- 월평균 수익률 > 5%
- BTC 대비 초과 수익 > +5%
- 연환산 수익률 > 60%
- 손실 한도 준수율 100%
- 거래 로그 무결성 100%

### Phase 5-1 (주식 시장)
- 월평균 수익률 > 7%
- KOSPI 대비 초과 수익 > +10%

### Phase 5-2 (멀티 거래소)
- 차익거래 성공률 > 80%
- 거래소 간 슬리피지 < 0.3%

### Phase 5-3 (AI 전략)
- AI vs 수동 전략 수익률 > +20%
- 샤프 비율 > 2.5
