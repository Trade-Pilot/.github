# Trade Pilot - 개발 로드맵

## 개발 전략

Trade Pilot은 **데이터 수집 → 시뮬레이션 → 가상거래 → 실거래** 순서로 점진적으로 개발합니다.

```
Phase 1          Phase 2          Phase 3          Phase 4
데이터 기반      전략 개발        실시간 검증      실전 투입
구축             및 검증          환경             자동화
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Market          Agent           VirtualTrade     Trade
Service         Service         Service          Service
(3개월)         (2개월)         (2개월)          (2개월)
```

## 우선순위 및 의존성

### 필수 선행 작업
```
Market Service (데이터 수집)
    ↓
Agent Service (전략 개발)
    ↓
Simulation Service (백테스팅)
    ↓
VirtualTrade Service (가상거래)
    ↓
Trade Service (실거래)
```

### 병렬 개발 가능 항목
- Frontend 개발 (각 Phase별 독립적 진행)
- Infra 구축 (Phase 1에서 완료)
- Exchange Service (Market과 병렬)

## 리스크 관리

### 기술적 리스크
- **데이터 누락**: Flat Candle로 보정, 수집 성공률 모니터링
- **시스템 장애**: HA 구성, 자동 재시작, 알림 시스템
- **API 제한**: Rate Limiting 준수, 백오프 전략

### 비즈니스 리스크
- **전략 실패**: 충분한 백테스팅, 가상거래 검증 필수
- **자금 손실**: 손실 한도 설정, 긴급 중단 시스템
- **규제 변경**: 거래소 정책 모니터링, 유연한 아키텍처

## 측정 지표 (KPI)

### Phase 1 (데이터 수집)
- 데이터 수집 성공률 > 99%
- 시스템 가동률 > 99.5%
- API 응답 시간 < 100ms (p95)

### Phase 2 (백테스팅)
- 전략 승률 > 60%
- 샤프 비율 > 1.5
- MDD < 20%

### Phase 3 (가상거래)
- 월간 수익률 > 시장 평균
- 일일 거래 실행 성공률 > 99%
- 리스크 한도 준수율 100%

### Phase 4 (실거래)
- 월간 수익률 > 5%
- 손실 한도 준수율 100%
- 거래 로그 무결성 100%
