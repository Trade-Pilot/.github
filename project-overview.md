# Trade Pilot - 프로젝트 개요

## 비전

Trade Pilot은 데이터 기반 자동 거래를 통해 안정적인 수익을 창출하는 자동 거래 플랫폼입니다.

### 핵심 가치
- **데이터 기반 의사결정**: 충분한 시장 데이터를 수집하고 분석하여 전략 수립
- **전략 검증**: 가상 환경에서 충분히 테스트한 후 실제 거래 적용
- **점진적 확장**: 수동 전략부터 시작하여 AI 기반 전략으로 진화
- **리스크 관리**: 시뮬레이션과 가상 거래를 통한 철저한 검증

## 주요 도메인

### 1. Market (시장 데이터 관리)

**책임**:
- 시장 데이터 수집 (OHLCV, 재무제표, 뉴스)
- 데이터 품질 관리 (누락 구간 보정)
- 다양한 시간 간격의 캔들 데이터 제공

**핵심 기능**:
- 거래 심볼 관리 (상장/폐지 추적)
- 캔들 데이터 수집 (1분봉 기준, 12개 간격)
- 데이터 조회 API 제공

**현재 상태**: ✅ 설계 완료 (OHLCV 수집 기능)

### 2. Agent (거래 주체)

**책임**:
- 시장 데이터 분석
- 매수/매도/관망 의사결정
- 전략 실행 및 성과 추적

**핵심 개념**:
- **Strategy**: 거래 전략 로직 (수동 작성 → AI 기반)
- **Signal**: 매수/매도 신호 생성
- **Portfolio**: 보유 자산 관리
- **Performance**: 성과 측정 및 분석

**개발 단계**:
1. Phase 1: 수동 전략 (기술적 지표 기반)
2. Phase 2: AI 전략 (강화학습)

**현재 상태**: 🔄 설계 필요

### 3. Simulation (시뮬레이션 환경)

**책임**:
- 과거 데이터 기반 백테스팅
- 전략 성능 검증
- 리스크 분석

**핵심 기능**:
- 시뮬레이션 환경 관리
- 과거 데이터 재생 (Time Travel)
- 성과 리포트 생성

**개발 단계**:
1. Phase 1: 단일 심볼 백테스팅
2. Phase 2: 멀티 심볼 포트폴리오 백테스팅
3. Phase 3: 병렬 시뮬레이션 (전략 최적화)

**현재 상태**: 🔄 설계 필요

### 4. VirtualTrade (가상 거래)

**책임**:
- 실시간 가상 거래 실행
- 실제 거래 전 최종 검증
- Paper Trading 환경 제공

**핵심 기능**:
- 실시간 시장 데이터 기반 거래
- 거래 수수료 및 슬리피지 시뮬레이션
- 실시간 성과 모니터링

**개발 단계**:
1. Phase 1: 단일 전략 가상 거래
2. Phase 2: 멀티 전략 동시 운영
3. Phase 3: 전략 자동 배포 시스템

**현재 상태**: 🔄 설계 필요

### 5. Trade (실제 거래)

**책임**:
- 실제 자금으로 거래 실행
- 주문 관리 (체결/취소)
- 실시간 포지션 관리

**핵심 기능**:
- 거래소 주문 실행
- 체결 내역 추적
- 리스크 관리 (손절/익절)

**개발 단계**:
1. Phase 1: 단일 거래소 연동 (업비트)
2. Phase 2: 멀티 거래소 지원
3. Phase 3: 고급 주문 유형 (조건부 주문)

**현재 상태**: 🔄 설계 필요

## 기술 스택

### Frontend
- **언어**: TypeScript
- **프레임워크**: React
- **아키텍처**: FSD (Feature-Sliced Design)

### Backend
- **언어**: Kotlin 2.0.21
- **프레임워크**: Spring Boot 3.4.0
- **아키텍처**: Microservice, DDD, Hexagonal Architecture
- **ORM**: JPA, QueryDSL

### Infrastructure
- **컨테이너**: Kubernetes (K3s)
- **CI/CD**: Jenkins, ArgoCD, Helm Chart
- **메시징**: Apache Kafka
- **데이터베이스**:
  - PostgreSQL (메인 DB)
  - TimescaleDB (시계열 데이터)
  - Redis (캐시)
- **모니터링**:
  - Elasticsearch, Kibana (로그)
  - Prometheus, Grafana (메트릭)

## 개발 원칙

### 1. 데이터 우선 접근
- 충분한 데이터 수집 없이 거래 시스템 개발하지 않음
- 최소 3개월 이상의 과거 데이터 확보 후 전략 개발

### 2. 점진적 개발
- 단순한 것부터 복잡한 것으로 (수동 전략 → AI 전략)
- 작은 자본으로 검증 후 확장 (시뮬레이션 → 가상거래 → 실거래)

### 3. 철저한 검증
- 모든 전략은 시뮬레이션 필수
- 가상 거래에서 2주 이상 안정적 성과 확인 후 실거래 적용

### 4. 리스크 관리
- 단일 거래 손실 한도 설정
- 일일/주간/월간 손실 한도 설정
- 자동 거래 중단 조건 명확화

## 지원 시장

### Phase 1: 암호화폐 (Coin)
- **거래소**: 업비트 (Upbit)
- **마켓**: KRW 마켓
- **이유**:
  - 24시간 거래 가능
  - API 접근성 우수
  - 높은 변동성 (전략 학습에 유리)

### Phase 2: 주식 (Stock)
- **거래소**: 한국투자증권 API
- **마켓**: KOSPI, KOSDAQ
- **추가 고려사항**:
  - 재무제표 데이터 수집
  - 거래 시간 제한
  - 공매도 규제

## 성공 지표

### 시스템 안정성
- 데이터 수집 성공률 > 99.9%
- API 응답 시간 < 100ms (p95)
- 시스템 가동률 > 99.5%

### 거래 성과

#### 수익률 목표
**Phase 2 (백테스팅)**:
- 시뮬레이션 승률: > 60%
- 샤프 비율: > 1.5
- 최대 낙폭 (MDD): < 20%

**Phase 3 (가상거래)**:
- 월평균 수익률: > 3%
- 벤치마크 대비 초과 수익: BTC 대비 +3%
- 최대 낙폭 (MDD): < 15%

**Phase 4 (실거래)**:
- 월평균 수익률: > 5%
- 벤치마크 대비 초과 수익: BTC 대비 +5%
- 연환산 수익률: > 60%
- 최대 낙폭 (MDD): < 15%

**Phase 5 (주식 확장)**:
- 주식 월평균 수익률: > 7%
- 벤치마크 대비 초과 수익: KOSPI 대비 +10%

#### 벤치마크 정의
**암호화폐 (COIN) - Phase 1~4**:
- **주요 벤치마크**: 비트코인 (BTC) 수익률
- **보조 벤치마크**:
  - 이더리움 (ETH) 수익률
  - 업비트 KRW 마켓 상위 10개 평균 수익률
- **목표**: BTC 수익률 대비 최소 +5% 초과 달성

**주식 (STOCK) - Phase 5**:
- **주요 벤치마크**: KOSPI 지수 수익률
- **보조 벤치마크**:
  - KOSDAQ 지수 수익률
  - 코스피 200 지수 수익률
- **목표**: KOSPI 수익률 대비 최소 +10% 초과 달성

#### 리스크 제한
- **손실 한도 준수율**: 100% (필수)
- 샤프 비율: > 1.5 (백테스팅), > 2.0 (실거래)
- 소티노 비율: > 2.0

#### 비교 방법
```
초과 수익률 = (전략 수익률 - 벤치마크 수익률)

예시:
- 전략 월 수익률: 8%
- BTC 월 수익률: 3%
- 초과 수익률: 5% ✅ (목표 달성)
```
